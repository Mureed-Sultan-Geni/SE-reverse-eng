Adding a new page

Let's start with some basics using what we already have in the build.

  • Create a new HTML or PUG file under demo/src/views/page/templates/site. To test things out copy demo/src/views/page/templates/site/home.html as a new file test.html.
  • Open that file and change the highlighted snippet below
        
From

        main(data-template='site/home' data-id='home')
      
        
To

        main(data-template='site/test' data-id='home')
  • Make a copy of demo/src/scripts/app/site/pages/home.js as test.js in the same folder.
  • Open the file and replace the highlighted snippets
        
From

        // demo/src/scripts/app/site/pages/home.js

export default {
  templates: ['site/home'],
  getComp: function() {
    return setjs.getComp('site/home');
  }
};
      
        
To

        // demo/src/scripts/app/site/pages/test.js

export default {
  templates: ['site/test'],
  getComp: function() {
    return setjs.getComp('site/test');
  }
};
  • Open demo/src/scripts/app/site/init.js and import the default export as test and add this line as the route handler, addPage('', test);, illustrated below with the comments alongside.
// demo/src/scripts/app/site/init.js

import {addPage, addPaths} from 'setbp/kernel/page-manager.js';

import './api.js';
import './template-functions.js';
import home from './pages/home.js';
import test from './pages/test.js';    // Added line

export default function() {
  addPage('', home);
  addPage('', test);                   // Added line

What we did

We created a new page associated with a route. Each page in SetJS has a template and typically an associated JS file (optional). The main power of SetJS comes from its focus on pages rather than components which is the norm these days. This is one of the greatest differences between major frameworks and SetJS.

A template file contains one or more elements with an attribute called “data-template” present. This becomes their identity name. The name can be anything, but a typical application uses the folder structure.
